字节码进阶

一、方法调用指令

方法调用共涉及五条指令：

  • invokestatic：用于静态方法调用
  • invokespecial：用于调用私有实例方法、构造器方法以及使用super关键字调用父类的实例方法等
  • invokevirtual：用于调用非私有实例方法
  • invokeinterface：用于调用接口方法
  • invokedynamic：用于调用动态方法

1. invokestatic

不需要将对象压栈，只需要将所需的参数压栈即可调用invokestatic

2. invokevirtual

调用目标方法在运行时根据对象的类型确认

需要将对象引用和方法参数入栈，方法调用结束时对象引用和方法参数都会出栈，返回值（如果有的话）会入栈

3. invokespecial

如前面说的new一个对象，就用到了invokespecial

4. invokeinterface

探究：与invokevirtual有什么不同？

  • 方法分派原理

    1. 单继承

      vtable: 虚方法表，子类的虚方法表保留了父类的虚方法表的顺序，invokevirtual调用时，只需要根据该方法在虚方法表中的数组索引（固定不变）找到对应的方法引用即可，十分高效

    2. 多接口

      itable：由偏移量（offset table）和方法表（method table）两部分组成，invokeinterface调用时：

      • 先在offet table中查找对应的方法表位置和方法位置（循环遍历：相对低效）
      • 再在method table中查找具体的方法实现（根据offset直接查找）

  • HSDB工具使用

    • sudo java -cp sa-jdi.jar sum.jvm.hotspot.HSDB
    • Class Browser
    • mem命令的使用

5. invokedynamic

  • MethodHandle — 方法句柄

    使用方法：

    1. 创建MethodType对象
    2. 调用MethodHandles.lookup方法，返回MethodHandles.Lookup对象，可以通过该对象查找指定MethodType的方法句柄，如：findStatic、findSpecial、findVirtual等
    3. 拿到方法句柄后，调用方法

  • invokedynamic执行流程

    1. jvm首次执行invokedynamic时，调用引导方法（Bootstrap Method）
    2. 引导方法返回一个CallSite对象，内部根据方法签名进行目标方法查找。它个getTarget方法返回方法句柄（MethodHandle）对象
    3. 在CallSite没有变化的情况下，MethodHandle可以一直被调用，如果CallSite有变化，重新查找即可

  • 真正执行lambda调用的还是用invokeinterface指令

3. synchronized实现原理

字节码层面有两种形态：

  • 使用synchronized修饰方法，方法的访问标记中 ACC_SYNCHRONIZED 标记为会被置为1
  • 使用synchronized修饰代码块，字节码中会出现：monitorenter monitorexit

每个java对象都可以作为一个实现同步的锁，这些锁被称为监视器锁（Monitor），有三种不同的表现形态：

  1. synchronized修饰非静态方法，监视器锁是当前对象
  2. synchronized修饰的静态方法，监视器锁是当前类的类对象
  3. synchronized(lock){} 同步代码块，监视器锁是lock对象

我们在观看synchronized(lock){}生成的字节码时，可以看到有一个monitorenter命令和两个monitorexit命令，这是因为:

jvm为了确保锁被释放，会自动生成异常处理器，相当于给同步代码块增加了try-finally
